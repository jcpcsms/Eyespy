EyeSpy+

Research Methods Matrix
NODE Research Problem Testable Hypothesis Method for Testing Planned Analyses Expected Outcome
01
Can the application detect and report an incident faster than a standard security monitoring system?  
EyeSpy+ reduces the amount of time it takes to report an incident by detecting firearms in plain sight and sending actionable notifications to stakeholders.
Incident simulations will be conducted in an open environment with modified nerf blasters and multiple participants that cause positive firearm detections in the application.
We will record the event (video) and incident response times from beginning to end. Then perform a case study of the metrics collected.
We expect the application to detect the mock firearms before the attacker is able to harm the other participants. Notifications should be received by the security team that enables them to send a text reply to escalate the alarm to a simulated law enforcement dispatcher.
02
Can the application recognize the intended signal for help gesture?
EyeSpy+ is able to create a security notification alert based on a pre-defined gesture.
Incident simulations will be conducted in multiple lighting conditions to test the capabilities of our gesture recognition feature. 
We will record the event (video) and incident response times from beginning to end. Then perform a case study of the metrics collected.
We expect the application to function as intended and distribute the system alert in the same manner as Node 1. 
03
Can the application handle multiple camera sources?
EyeSpy+ is able to successfully perform real-time object detections and gesture recognition tasks while multiple camera feeds are being monitored.
Incident simulations will be conducted in an open environment with modified nerf blasters and multiple participants that cause positive firearm detections in the application. We will add as many camera feeds as possible to stress test the application and hardware limits. We will also test with a single video source that contains a security camera systems matrix view, (multiple cameras as a single source) to evaluate detections on this type of video source.
We will record video of the applications performance and host system metrics to determine the limits of its capability as a desktop application. 
We expect there to be a necessary trade off in frames per second as additional sources are added. We expect the single video source security camera systems matrix to miss detections and detect with lower confidence scores.
